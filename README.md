### 저의 작업 내역

- **[가상 리스트를 활용한 렌더링 최적화]**
    - **배경:** `React-window` 라이브러리를 이용하여 무한 스크롤 페이지를 가상 리스트로 구현하게 된 배경은, 대량의 데이터를 처리하는 데 필요한 메모리 사용을 개선하고자 하는 목표로 인해 시작되었습니다.
    - **목표:** 주어진 상황 속에서 초기 렌더링 속도의 개선과 사용자 경험을 향상시키는 것이 주된 목표였습니다.
    - **행동:** 스크롤 이벤트의 부하를 줄이기 위해 쓰로틀링 처리를 적용하고, 가상 목록에서 데이터를 동시에 가져오고 동시 스크롤을 활성화하여 무한 스크롤 구현에서 발생하는 빈번한 레이아웃 변경 필요성을 제거하는 등의 행동을 취했습니다.
    - **이유:** 이러한 행동을 취한 이유는 초기 렌더링 속도를 개선하고 사용자 경험을 향상시키기 위해 메모리 사용을 최적화하고 성능을 향상시키기 위해서였습니다.
    - **회고:** 이 프로젝트를 통해 가상 리스트와 무한 스크롤에 대한 기본적인 개념과 사용 방법을 익히게 되었으며, 효과적인 스크롤 관리와 데이터 렌더링을 위한 최적화 방법을 배웠습니다. 앞으로는 대규모 데이터 처리와 렌더링에 유용하게 활용할 계획입니다.
    

- **[가상 리스트와 브라우저 스크롤 동기화 작업]**
    - **배경:** 윈도우와 가상 리스트 스크롤 분리 현상은 프로젝트를 진행하던 중 발생한 문제였습니다. 이는 가상 리스트의 높이 값과 브라우저 내의 전체 높이 값이 다르기 때문에 생긴 문제였습니다. 이러한 상황 속에서 해당 라이브러리가 스크롤을 통합하는 기능을 지원하지 않았습니다.
    - **목표**: 이 문제를 해결하기 위한 목표는 윈도우 스크롤과 콘텐츠 스크롤의 분리 현상을 해소하고, 원활한 스크롤 작동을 보장하는 것이었습니다.
    - **행동**: `react-window`의 스크롤을 추적하기 위해 `useRef`를 활용하여 해당 DOM의 위치를 감시했습니다. scrollTo 함수와 scrollOffset을 활용하여 두 스크롤의 동기화를 조정하고, `position: absolute`를 이용하여 `react-window`의 내부 동작을 이해하고 조작했습니다.
    - **이유**: 스크롤을 통합하여 윈도우 스크롤과 콘텐츠 스크롤 간의 분리 현상을 제거함으로써 라이브러리의 적절한 작동을 가능하게 하기 위함이었습니다.
    - **회고**: 이 과정을 통해 라이브러리의 내부 동작과 스크롤 연동 방법을 보다 깊이 이해할 수 있었으며, 이후 유사한 상황에서 빠르고 효율적으로 대처할 수 있는 능력을 갖추게 되었습니다.

- **[API 요청 및 초기 렌더링 최적화]**
    - **배경** : 전체 1000개의 API 요청 시 초기 렌더링이 3500ms까지 지연되는 문제로 인해 사용자 경험이 저하되었습니다.
    - **목표** : 초기 렌더링을 2000ms 내에 완료하고 나머지 전체 콘텐츠를 빠르게 불러와 사용자 경험을 향상시키는 것이 목표였습니다.
    - **이유:** 초기 렌더링을 빠르게 완료함으로써 사용자들이 기다리지 않고 콘텐츠를 볼 수 있게 하여 전반적인 사용자 경험을 향상시키고자 했습니다.
    - **행동:** 초기 렌더링 API 요청과 후속 API 요청을 분할하여 처리하고, 초기 렌더링이 완료된 후 나머지 전체 API를 즉시 비동기적으로 요청하여 렌더링 속도를 최적화했습니다.
    - **회고:** 위의 변경 사항을 통해 초기 렌더링 속도를 1200ms 까지 줄여 사용자 경험을 개선하고 전체적인 웹 애플리케이션의 성능을 향상시켰습니다.
